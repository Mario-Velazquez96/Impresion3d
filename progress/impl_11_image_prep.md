## Decisions / deviations (all minor, none change spec behavior)

1. **`mapToPalette(src, palette, dither)` exported** in addition to the
   spec-listed `quantize`. Reason: the tasks.md test "a hand-computed
   Floyd–Steinberg case (2×2 grey vs black/white palette)" is mathematically
   impossible through `quantize` alone — a median-cut palette derived from an
   image containing grey can never be exactly {black, white} (box means
   include the grey). `quantize` is now the trivial composition
   `mapToPalette(src, medianCutPalette(src, n), dither)`. Pure superset; no
   behavior change.
2. **Validation + decode invocation live in `ImageDropzone`** (design.md
   sketched the decode call in the island). Keeps the `role="alert"` error
   region single-sourced (type/size/decode failures all report in one place)
   and automatically leaves island state untouched on failure (R3). The
   island still owns all pipeline state; decode glue itself is the separate
   mockable `decode.ts` module.
3. **`worker-messages.ts` carries the two tiny (de)serialize helpers** (design
   said "types only"). Both the worker and the island need them; sharing
   prevents wire-shape drift. Fully unit-covered (100%).
4. **Classify-boundary test is near-threshold, not exactly-at-threshold**: an
   HSL saturation of exactly 0.12 is not representable in float from 8-bit
   channels (every s is a ratio of /255 fractions), so the strict-`<`
   comparison is pinned by the nearest representable neighbors
   (s≈0.102 → neutral, s≈0.124 → color) with a comment explaining why.
5. **Posterize is allowed straight from the `loaded` stage** (adjust is
   optional per the requirements' "Adjust (optional…)"): the original then
   serves as the adjusted source and the histogram slot is null until the
   first Apply. R7's "quantize the adjusted image" is preserved — after any
   Apply, posterize always consumes the adjusted buffer.
6. Extra tests beyond tasks.md: hook tests against a stubbed global `Worker`
   and decode tests against stubbed `createImageBitmap`/canvas — added to hit
   the ≥80%-lines target on those changed modules (they'd otherwise be
   covered only by E2E).

palette-cleanup edits**. Reverts merge / merge-similar / merge-tiny / snap one
step at a time back to the fresh-posterize palette. No Redo. No worker,
`lib/image-prep-core.ts`, schema, dependency, or persistence change.

### Files modified

- `components/image-prep/ImagePrep.tsx` — added a bounded palette undo history
  **inside the `quantized` stage** (`history: { image; preview }[]`, cap
  `MAX_PALETTE_HISTORY` = 20, oldest dropped). Posterize seeds the baseline;
  each palette action pushes the new state; `handleUndo` is a **pure client pop**
  (functional `setStage`, no worker request, no recompute) that restores the

## Enhancement: multi-select palette merging (R22, supersedes tap-two R10) — 2026-07-18

Replaced the instant tap-to-merge with an explicit **multi-select** model:
tapping a swatch (or picking a pixel, R21) toggles it in/out of a selection;
an action bar merges the whole selection. No schema, dependency, env var, or
persistence change; quantize/adjust logic untouched.

### What changed

- `lib/image-prep-core.ts` — two new pure functions, core still **100%
  branch**: `mergeManyEntries(image, from[], into)` (dedupes `from`, ignores
  `into` inside it, remaps + sums counts, survivor keeps color AND catalog,
  no-op same-reference when nothing left) and
  `mergeEntriesToAverage(image, indices[])` (rounded count-weighted average
  RGB at the LOWEST selected index, survivor's `catalog` cleared, `< 2`
  distinct indices no-op, unweighted-mean fallback when every selected count
  is zero).
- `components/image-prep/worker-messages.ts` +
  `image-prep.worker.ts` — `PaletteAction` swaps the now-unreachable
  `{ kind: "merge" }` for `{ kind: "mergeMany" }` and
  `{ kind: "mergeAverage" }`, dispatched to the new core functions with the
  same preview regeneration — so both merges ride the R20 undo history for
  free.

  `number[]` with a shared `toggleSelected` (swatch taps + eyedropper, which
