## Verification

- `/finances` is reachable by any signed-in user from the nav (app **and** admin); signed-out redirects to `/login` (R1).
- Worked example: sales `1250.00 + 0.10 + 0.20 + 99.95` = **$1,350.25**, withdrawals `500.00 + 350.25` = **$850.25**, balance **$500.00** — and a **$2,000.00 expense changes nothing** (R2, R3, R7).
- The balance is labelled "Sales minus withdrawals — does not include expenses" (R4); empty ledgers show **$0.00** (R5); sales < withdrawals shows **-$150.50**, signed and unclamped (R6).
- No balance column in the schema; no `toNumber`/float in the money path; totals come from `_sum` (R2, R7).
- A sale requires a print (R8); a print with sales cannot be deleted (R9).
- Any user records sales; sale delete, withdrawal record, and withdrawal delete are **Admin-only, server-enforced, with no write on rejection** (R10, R11, R12); unauthenticated mutations write nothing (R13); negative/invalid amounts are rejected and clamped (R14).
- Withdrawals carry a server-assigned `recordedBy` shown in the list (R15); RLS is enabled + forced on both tables (R16); both ledgers list date-desc in one query each (R17).
